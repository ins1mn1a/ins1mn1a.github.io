自定义阻塞队列，线程池

自定义线程池

池化技术有很多, 比如线程池、数据库连接池、HTTP连接池等等都是对这个思想的应用。池化技术的思想主要是为了减少每次获取资源的消耗，提高对资源的利用率。

  • 降低资源消耗

    通过重复利用已创建的线程降低线程创建和销毁造成的消耗。(创建的线程，实际最后要和操作系统的线程做映射，很消耗资源)

  • 提高响应速度

    当任务到达时，任务可以不需要等到线程创建就能立即执行。

  • 提高线程的可管理性

    线程是稀缺资源，如果无限制的创建，不仅会消耗系统资源，还会降低系统的稳定性，使用线程池可以进行统一的分配，调优和监控。

阻塞队列

阻塞队列就是存放一系列的任务，当有任务需要完成的时候，并且线程池中各个线程正忙着执行自己的任务，这时候main线程或者其他线程存放其任务就存放在阻塞队列中，线程池中的线程从阻塞队列中取任务执行，别的线程把任务放到线程池的阻塞队列中等待执行。

阻塞队列的成员

  • 整个阻塞队列首先需要一个队列用来存放阻塞的任务。

    private Deque<T> deque=new ArrayDeque<>();

  • 需要一个锁对象用来解决线程安全问题，这里采用ReentrantLock。

    private ReentrantLock lock=new ReentrantLock();

  • 任务生产者的条件变量Condition，阻塞队列满就不能再生产

    private Condition fullWaitSet = lock.newCondition();

  • 任务消费者即线程池中的核心线程的条件变量Condition，阻塞队列空就不能再消费

    private Condition emptyWaitSet = lock.newCondition();

  • 阻塞队列的容量，最多可以容纳多少个任务，能同时存在的任务=线程数+队列容量

    private int capacity;

阻塞队列的方法

  • public T take()，用于线程消费任务
  • public T poll(long timeout, TimeUnit unit)，增强的take()方法，take方法在等待阻塞队列为空的时候会一直等待，poll方法可以设置超过一定时间不再等待
  • public void put(T element)，生产者往阻塞队列里放任务的方法

线程池

线程池的成员

  • 阻塞任务队列

    private BlockingQueue<Runnable> taskQueue;

  • 核心线程数

    private int coreSize;

  • 线程集合

    private HashSet<Worker> workers=new HashSet<>(coreSize);

    • 每个Worker都是一个线程，Worker继承Thread，有一个Runnable成员变量，用来接收任务

      private Runnable task;

  • 获取任务的超时时间

    private long timeout;

  • 时间单位

    private TimeUnit unit;

线程池的方法

  • 线程池开始工作，不停的给线程池传递任务

    public void execute(Runnable task)

再新建5个任务，任务就是打印一下 i 的值

运行结果

  • 可以看到先新建了两个任务@2aaf7cc2，@23223dd8，execute后就新建了两个线程
  • 又新建了一个任务@1b40d5f0，但是两个线程正忙，所以进入阻塞队列
  • 最开始新建的两个任务打印完毕，1,0
  • Thread-0执行完了之后从阻塞队列中取到了任务@1b40d5f0，然后执行
  • 接下来main线程又生产了两个任务@ea4a92b，@3c5a99da加入阻塞队列
  • Thread-0执行3，Thread-1执行4
  • 程序阻塞了，因为主线程的5个任务已经执行完成了，但是Thread-0，Thread-1仍然阻塞于获取任务的take()方法之中，while (task!=null||(task = taskQueue.take())!=null)；因此，要给线程一个超时时间，等待一会没有任务后，就结束线程
  • 接下来测试线程在从阻塞队列中获取任务的时候采用poll(long timeout,TimeUnit unit)方法，这样线程在没有任务可以获取到的时候会自动结束任务
